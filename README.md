## 2. 시나리오 (5~10줄)

<br>
이 프로그램은 현대 사회의 경제 활동에 필수적인 **'개인별 신용 등급'**을 체계적으로 판독하고 관리하는 스마트 금융 시스템입니다. 
가장 먼저 시스템을 방문한 사용자에게 이름과 현재 연봉, 보유하고 있는 부채(빚), 그리고 지난 한 해 동안의 연체 횟수와 같은 금융 데이터를 정중하게 물어보고 그 대답을 변수에 기록하는 것으로 서비스가 시작됩니다. 그다음에는 사용자가 대답한 수치들을 금융 공학적 로직에 대입하여 이 사용자의 경제적 신뢰도가 어느 정도인지 **'최종 신용 점수'**로 환산해 봅니다. 이렇게 계산된 신용 점수가 기준치보다 높으면 우량 고객인 '1등급' 판정을 내리고 대출 우대 혜택을 안내하지만, 점수가 낮으면 주의가 필요한 '저신용' 등급으로 분류하여 개선 권고를 해줍니다. 만약 연봉이 매우 높으면서도 연체 횟수가 0회인 아주 성실한 고객이 있다면, **'VVIP 마스터'**라는 특별한 칭호와 함께 특별 한도 상향이라는 깜짝 혜택까지 제공할 계획입니다. 나중에는 한 명의 데이터만 처리하는 것이 아니라, 우리 시스템을 거쳐 간 **수많은 고객의 정보를 커다란 데이터 주머니(리스트)**에 차곡차곡 담아서 관리하고, 관리자가 원할 때마다 전체 고객 명부와 등급 현황을 한눈에 꺼내 볼 수 있게 만들 것입니다. 마지막으로 이 모든 기능은 사용자가 화면에 뜨는 메뉴판에서 번호를 눌러 자유롭게 선택할 수 있고, **'시스템 종료'**를 선택하기 전까지는 계속해서 새로운 고객의 등급을 판독할 수 있는 전문적인 콘솔 기반 금융 소프트웨어로 완성될 계획입니다.


## 3. 예상 기능 및 메뉴 (최소 5개)
1. 신규 고객 정보 등록
2. 최종 신용 점수 산출
3. 신용 등급 판정 및 결과 확인
4. VVIP 특별 칭호 및 한도 부여
5. 대출 적격성 자동 심사
6. 전체 고객 데이터 명부 출력
7. 특정 고객 정보 검색 및 관리
0. 시스템 업무 종료

적용 내용: AI와 상담하며 단순한 계산기를 넘어 **'금융 데이터 기반의 신용 등급 관리'**라는 명확한 비즈니스 컨셉을 설정함. 이를 통해 입출력부터 배열 관리까지 이어지는 4단계 로직의 일관성을 확보하고, 실제 프로그램다운 구조적 기틀을 마련함.

2. 내용 2 : 과제 요구사항에 최적화된 시나리오 구성

프롬프트 요약: "제시된 길드 관리 시스템 예시의 톤앤매너를 유지하면서, 신용등급 시스템에 맞게 4단계 개발 로드맵이 포함된 상세 시나리오 작성을 요청함."

적용 내용: 사용자가 데이터를 입력하는 순간부터 등급 판정, 리스트 저장, 메뉴 반복 실행까지의 전 과정을 '금융 소프트웨어 서비스' 관점에서 서사적으로 풀어냄. 이를 통해 코딩의 목적(누가, 왜, 어떻게 사용하는가)을 분명히 하여 과제의 완성도를 높임.

3. 내용 3 : 확장성을 고려한 7가지 핵심 메뉴 설계

프롬프트 요약: "기본적인 등급 판독 외에 프로그램의 풍성함을 위해 최소 5개 이상의 구체적인 기능과 메뉴 구성을 제안해달라고 요청함."

적용 내용: 단순 등급 출력을 넘어 'VVIP 특별 칭호 부여', '대출 적격성 심사', '전체 고객 명부 검색' 등 실제 은행 시스템에서 쓰일 법한 7가지 기능을 메뉴화함. 특히 리스트와 함수가 필요한 4단계 심화 기능을 미리 배치하여 추후 확장 개발이 용이하도록 설계함.

🛠️ Trouble Shooting (예상 문제 및 해결 방안)
1. 입력값 자료형 오류 (ValueError)
문제 발생: input()으로 받은 값은 기본적으로 **문자열(str)**인데, 이를 숫자로 변환하지 않고 score = 500 + salary 처럼 계산하려고 하면 에러(TypeError)가 발생함. 혹은 숫자를 입력해야 하는데 사용자가 실수로 "만원" 같은 글자를 포함해 입력하면 프로그램이 강제 종료됨.

해결 방법: int(input())으로 형변환을 명확히 해주고, 보고서에는 "데이터 연산을 위해 문자열 데이터를 정수형으로 캐스팅하여 해결함"이라고 작성하면 좋아.

심화 해결: try-except 구문을 사용해 숫자가 아닌 값이 들어오면 "숫자만 입력해 주세요"라고 안내하도록 예외 처리를 추가함.

2. 데이터 미등록 상태에서의 접근 오류
문제 발생: 프로그램을 켜자마자 1번(등록)을 수행하지 않고, 2번(조회)이나 5번(출력)을 먼저 누르면 리스트가 비어 있어 에러가 발생하거나 텅 빈 화면이 나옴.

해결 방법: 각 메뉴의 시작 부분에 if not customer_list: 조건문을 추가하여, 데이터가 없을 경우 "등록된 데이터가 없습니다"라는 안내 문구를 출력하고 메뉴로 되돌아가게 처리함(방어적 프로그래밍).

3. 점수 범위 초과 문제 (Out of Range)
문제 발생: 연봉이 엄청나게 높거나 부채가 너무 많을 경우, 신용 점수가 1000점을 넘어가거나 마이너스(-) 점수가 나오는 현상 발생. 현실적인 등급 산출에 어긋남.

해결 방법: 점수 계산 로직 하단에 if score > 1000: score = 1000, if score < 0: score = 0 처럼 **최댓값과 최솟값의 한계치(Boundary)**를 설정하는 코드를 추가하여 데이터의 신뢰성을 높임.
