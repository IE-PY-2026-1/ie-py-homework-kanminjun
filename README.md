## 2. 시나리오 (5~10줄)

<br>
이 프로그램은 현대 사회의 경제 활동에 필수적인 **'개인별 신용 등급'**을 체계적으로 판독하고 관리하는 스마트 금융 시스템입니다. 
가장 먼저 시스템을 방문한 사용자에게 이름과 현재 연봉, 보유하고 있는 부채(빚), 그리고 지난 한 해 동안의 연체 횟수와 같은 금융 데이터를 정중하게 물어보고 그 대답을 변수에 기록하는 것으로 서비스가 시작됩니다. 그다음에는 사용자가 대답한 수치들을 금융 공학적 로직에 대입하여 이 사용자의 경제적 신뢰도가 어느 정도인지 **'최종 신용 점수'**로 환산해 봅니다. 이렇게 계산된 신용 점수가 기준치보다 높으면 우량 고객인 '1등급' 판정을 내리고 대출 우대 혜택을 안내하지만, 점수가 낮으면 주의가 필요한 '저신용' 등급으로 분류하여 개선 권고를 해줍니다. 만약 연봉이 매우 높으면서도 연체 횟수가 0회인 아주 성실한 고객이 있다면, **'VVIP 마스터'**라는 특별한 칭호와 함께 특별 한도 상향이라는 깜짝 혜택까지 제공할 계획입니다. 나중에는 한 명의 데이터만 처리하는 것이 아니라, 우리 시스템을 거쳐 간 **수많은 고객의 정보를 커다란 데이터 주머니(리스트)**에 차곡차곡 담아서 관리하고, 관리자가 원할 때마다 전체 고객 명부와 등급 현황을 한눈에 꺼내 볼 수 있게 만들 것입니다. 마지막으로 이 모든 기능은 사용자가 화면에 뜨는 메뉴판에서 번호를 눌러 자유롭게 선택할 수 있고, **'시스템 종료'**를 선택하기 전까지는 계속해서 새로운 고객의 등급을 판독할 수 있는 전문적인 콘솔 기반 금융 소프트웨어로 완성될 계획입니다.


## 3. 예상 기능 및 메뉴 (최소 5개)
1. 신규 고객 정보 등록
2. 최종 신용 점수 산출
3. 신용 등급 판정 및 결과 확인
4. VVIP 특별 칭호 및 한도 부여
5. 대출 적격성 자동 심사
6. 전체 고객 데이터 명부 출력
7. 특정 고객 정보 검색 및 관리
0. 시스템 업무 종료

적용 내용: 사용자가 데이터를 입력하는 순간부터 등급 판정, 리스트 저장, 메뉴 반복 실행까지의 전 과정을 '금융 소프트웨어 서비스' 관점에서 서사적으로 풀어냄. 이를 통해 코딩의 목적(누가, 왜, 어떻게 사용하는가)을 분명히 하여 과제의 완성도를 높임.

프롬프트 요약: "기본적인 등급 판독 외에 프로그램의 풍성함을 위해 최소 5개 이상의 구체적인 기능과 메뉴 구성을 제안해달라고 요청함."

적용 내용: 단순 등급 출력을 넘어 'VVIP 특별 칭호 부여', '대출 적격성 심사', '전체 고객 명부 검색' 등 실제 은행 시스템에서 쓰일 법한 7가지 기능을 메뉴화함. 특히 리스트와 함수가 필요한 4단계 심화 기능을 미리 배치하여 추후 확장 개발이 용이하도록 설계함.

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
개인별 금융 데이터 등록 및 신용 등급 정밀 판독 시스템 구현으로, 사용자로부터 경제 지표 데이터를 입력받아 리스트에 저장하고, 금융 공학적 가중치 연산을 통해 최종 신용 점수를 도출하는 시스템을 구축하는 것입니다. 특히 if-elif 조건문과 and/or 논리 연산자를 활용하여 신뢰 등급을 분류하고, 특정 조건을 만족하는 우량 고객에게 **'VVIP 마스터'**와 같은 특별 칭호를 부여하는 지능형 금융 로직을 완성하는 데 중점을 두었습니다.데이터 수집: input()으로 고객의 이름, 연봉, 총 부채, 연체 횟수를 입력받아 리스트에 저장.신용 점수 산출: 금융 가중치 공식을 적용하여 종합 신용 점수 산출:$Score = ( 연봉 \times 0.6 ) - ( 부채 \times 0.25 ) - ( 연체 횟수 \times 150 )$등급 판정: if-elif-else를 사용하여 1등급(우량)부터 5등급(위험)까지 5단계 등급 판정 로직 추가.특별 혜택 부여: and 연산자를 활용하여 '연봉 1억 이상'이면서 '연체 횟수 0회'인 경우 'VVIP 마스터' 칭호와 한도 상향 혜택 부여 기능 구현.

🛠️ Trouble Shooting (예상 문제 및 해결 방안)
1. 입력 값의 형변환(Casting) 오류

문제: input()으로 받은 연봉과 부채가 문자열(str)이라서 가중치 곱셈 연산 시 에러가 발생하거나 글자가 반복되는 현상이 나타남.

해결: int(input())을 사용하여 입력 단계에서 즉시 정수형으로 변환하여 산술 연산이 가능하도록 해결함.

2. 논리 연산자의 우선순위 혼동

문제: VVIP 조건을 체크할 때 연봉 > 10000 or 연체 == 0으로 작성하여, 연봉만 높으면 연체가 많아도 VVIP가 되는 논리적 허점이 발견됨.

해결: 두 조건을 모두 만족해야 하는 금융 원칙에 따라 or를 and 연산자로 수정하여 엄격한 심사 로직을 구현함.

3. 점수 하한선 및 상한선 설정

문제: 부채가 너무 많으면 신용 점수가 마이너스(-)가 나오는 비현실적인 상황 발생.

해결: if score < 0: score = 0 로직을 추가하여 점수가 0점 밑으로 떨어지지 않게 보정하는 '데이터 정규화' 과정을 거침.
